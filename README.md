## 🎨 Дизайн-система

### Nordic Layered Vector Theme

- **Философия:** Минимализм, слоистая глубина, векторная четкость
- **Шрифты:** Inter (заголовки), Inter (текст)
- **Цвета:** Deep Blue #1B3A5F + Amber #D49B3B
- **Стили:** `src/styles/nordic-tokens.scss`, `src/styles/nordic-base.scss`

**Палитра:**
- **Deep Blue (#1B3A5F)** — основной цвет бренда
- **Amber (#D49B3B)** — акцентный цвет
- **Deep Blue Light (#254D7A)** — светлый вариант основного
- **Amber Dark (#B8862B)** — темный вариант акцента
- **Neutral Grays** — минималистичная основа для фонов и текста

**Концепция:** Скандинавский дизайн в сочетании с современной веб-инженерией. Минимализм с глубиной слоев, ясность векторных форм, функциональность превыше формальностей.

### 🎭 Цвета категорий (Market Lab Characters - Nordic Palette)

Каждая категория закреплена за персонажем Market Lab в Nordic palette:

| **Kai** 🔵 | `technical-analysis` | Darker Blue | `#142A42` | `.cat-technical-analysis` |

**Почему Nordic palette:**
- ✅ Минималистичность и чистота
- ✅ Высокий контраст для читаемости
- ✅ Логичная иерархия цветов
- ✅ Соответствует современным трендам
- ✅ Поддержка light/dark тем

**Использование:**
```css

  background-color: var(--cat-crypto); /* Или другая категория */
  color: var(--nordic-text-primary); /* Темный текст на светлых фонах */
}
```

## 🌍 Интернационализация

Блог поддерживает 1 язык:
- 🇷🇺 Русский (RU)

Контент хранится в `src/content/blog-[lang]/`

**Структура статьи:** одна папка = одна статья; внутри только `index.md` (и при необходимости `assets/` для картинок). Frontmatter: `title`, `description`, `pubDate`, `category`. Поле **`tags` не используется** — только категория.

**📝 Руководство по написанию статей:** [`docs/CONTENT_WRITING_GUIDE.md`](docs/CONTENT_WRITING_GUIDE.md) — структура, размер, читаемость, SEO, чек-лист.

**Рабочая ветка:** `main`. Клонирование и pull — с `origin main`.

## 📝 Контент

Статьи пишутся в Markdown формате в папке `src/content/blog-[lang]/[slug]/`

**Структура статьи:**
```markdown
---
title: "Заголовок"  # До 100 символов, H1
description: "Описание (150-160 символов для SEO)"
pubDate: '2026-01-15'  # Формат: YYYY-MM-DD
category: 'crypto'  # Одна из 5 категорий
---

# Заголовок статьи

Контент статьи...

**Примечание:**
- Баннеры генерируются автоматически через компонент `<Banner>`
- `readingTime` вычисляется автоматически (слова / 200)
- Для OG тегов используется `title` (metaOgTitle не используем)

**OG изображения:** Автоматически генерируются при сборке (`npm run build`) в `dist/og/` для превью в соцсетях и Telegram. См. `docs/OG_IMAGES_MECHANICS.md` для подробностей.
```

## 🛠️ Технологии

- **Astro** — статический генератор сайтов
- **TypeScript** — типизация
- **Markdown** — контент блога

- **MD5 hashing** — умная инвалидация OG изображений

## 🔒 Безопасность

**⚠️ ВАЖНО:** В публичный репозиторий коммитятся только файлы, необходимые для функционирования блога.

- Планы, стратегии и внутренняя документация хранятся в [приватном репозитории](https://github.com/Hampfree-hub/hampfreeblog-private)
- Файлы планов, процедур, баннеров и т.п. перечислены в `.gitignore`. Список путей для синхронизации в private хранится **только в приватном репо** (не в публичном)
- API ключи и секреты не должны попадать в репозиторий
- **Коммиты:** сообщения — на английском (кодировка). Чтобы не светить личную почту в истории, в этом репо задать: `git config user.email "Hampfree-hub@users.noreply.github.com"` (аккаунт Hampfree-hub на GitHub)

## 📍 Деплой

Блог автоматически деплоится на GitHub Pages через GitHub Actions при пуше в ветку `main`.

### OG изображения

OG изображения автоматически генерируются при сборке и деплоятся вместе с сайтом:
- **Генерация:** `scripts/generate-og-images.mjs` (запускается после `npm run build`)
- **Хранение:** `dist/og/` (не коммитится, генерируется при сборке)
- **Доступ:** `https://hampfree-hub.github.io/Felag-academy/og/{lang}-{slug}.png`
- **Использование:** Автоматически подтягиваются в Telegram/соцсетях через meta tags

Подробнее: `docs/OG_IMAGES_MECHANICS.md` и `docs/OG_IMAGES_TELEGRAM.md`

## 📊 Аналитика

### GA4 Отслеживание

- **Событие:** `referral_banner_click` (клики по реферальным баннерам)
- **Параметры:** `banner_id`, `banner_name`, `article_category`, `article_lang`, `article_path`
- **UTM-метки:** Автоматически добавляются ко всем реферальным ссылкам
- **Документация:** Приватный репо → `docs/04-reports/2026-03/GA4_BANNER_TRACKING.md`

### Реферальные баннеры

- **Автовставка:** После 2-го H2 в каждой статье
- **Компонент:** `BannerCard.astro` (ручная вставка в MDX)
- **Конфигурация:** `src/config/ad-banners.ts`
- **Баннеры:** Veles, Bybit, BingX, Bitget, FinBazar, Default
