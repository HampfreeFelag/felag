# 📱 Telegram Preview: Как обновить кэш OG изображений

**Проблема:** После деплоя новых статей или обновления OG изображений Telegram может показывать старую версию превью (чёрный фон, старое название, старую категорию).

**Причина:** Telegram кэширует OG метаданные (title, description, image) навсегда и не обновляет их автоматически.

---

## 🔧 Решение: Обновление кэша через @WebpageBot

### Способ 1: Через Telegram бота (рекомендуется)

1. Открой [@WebpageBot](https://t.me/WebpageBot) в Telegram
2. Отправь команду `/new` или просто ссылку на статью:
   ```
   https://Felag-academy.org/ru/library/bitcoin-etf-2026-guide/
   ```
3. Бот ответит: "Preview updated" — кэш обновлён!

**Что происходит:**
- Telegram заново запрашивает OG метаданные с твоего сайта
- Скачивает новое OG изображение
- Обновляет кэш для всех будущих отправок ссылки

### Способ 2: Массовое обновление (несколько статей)

Отправь боту список ссылок (каждую с новой строки):
```
https://Felag-academy.org/ru/library/bitcoin-etf-2026-guide/
https://Felag-academy.org/ru/library/ethereum-basics-2026/
https://Felag-academy.org/en/library/solana-overview-2026/
```

## 🛠️ Локальная проверка OG изображений

Перед деплоем можно проверить, как будут выглядеть OG изображения:

### 1. Проверить наличие файла
```bash
# Windows PowerShell
ls dist/og/ru-bitcoin-etf-2026-guide.png

# Linux/Mac
ls dist/og/ru-bitcoin-etf-2026-guide.png
```

### 2. Открыть файл локально
```bash
# Windows
start dist/og/ru-bitcoin-etf-2026-guide.png

# Linux/Mac
open dist/og/ru-bitcoin-etf-2026-guide.png
```

### 3. Проверить HTML метатеги
Открой `dist/ru/library/bitcoin-etf-2026-guide/index.html` и найди:
```html
<meta property="og:image" content="https://Felag-academy.org/og/ru-bitcoin-etf-2026-guide.png">
```

## ⚠️ Troubleshooting

### Проблема: Telegram показывает чёрную карточку

**Причина 1:** OG изображение не сгенерировано
```bash
# Проверить наличие
ls dist/og/ru-{slug}.png

# Сгенерировать заново
npm run generate-og
```

**Причина 2:** Telegram кэширует старую версию
→ Отправь ссылку в @WebpageBot

**Причина 3:** Неправильный URL в meta tag
→ Проверь `astro.config.mjs`, что `site: 'https://Felag-academy.org'`

### Проблема: Долгая генерация OG изображений

Генерация ~170 статей занимает 3-7 минут. Это нормально.

**Для ускорения:**
- GitHub Actions кэширует зависимости
- Puppeteer использует системный Chromium (не качает каждый раз)

### Проблема: @WebpageBot не обновляет кэш

1. Подожди 1-2 минуты после деплоя (сайт должен быть доступен)
2. Проверь, что OG изображение доступно по прямому URL:
   ```
   https://Felag-academy.org/og/ru-bitcoin-etf-2026-guide.png
   ```
3. Попробуй команду `/clear` в @WebpageBot, затем отправь ссылку снова

## 📅 Когда обновлять кэш?

| Сценарий | Нужно ли обновлять? |
|----------|---------------------|
| Новая статья | ✅ Да |
| Обновление заголовка статьи | ✅ Да |
| Обновление описания статьи | ✅ Да |
| Обновление OG изображения | ✅ Да |
| Изменение дизайна OG шаблона | ✅ Да (для всех статей) |
| Технический деплой (без изменений контента) | ❌ Нет |

## 🎯 Best Practices

1. **Всегда обновляй кэш после деплоя** новых статей
2. **Проверяй превью** перед отправкой в публичные каналы
3. **Используй `npm run build:full`** для продакшен деплоя
4. **Сохраняй эту инструкцию** в закладки
